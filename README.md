# Horse Data Visualizer

『ウイニングポスト』で管理している生産馬データを、CSV の取り込み・編集・集計・可視化で扱うための Windows 向け Flutter アプリです。

生産馬の一覧確認だけでなく、種牡馬・繁殖牝馬・系統単位での集計、年次ごとの入力補助、マスタ編集までを 1 つのアプリ内で行えます。

## 対象ユーザー

* 『ウイニングポスト』をプレイしていて、生産馬データを整理したい人
* CSV で管理している種牡馬 / 繁殖牝馬 / 生産馬データを見やすくしたい人
* 系統確立や零細系統保護のために、生産状況をまとめて確認したい人

## 動作環境

* Windows
* Flutter / Dart 実行環境があること（ソースから起動する場合）

他環境での動作は未確認です。

## 主な機能

### 1. CSV インポート / エクスポート

ホーム画面から以下の CSV を取り込めます。

* 生産馬 CSV
* 種牡馬 CSV
* 繁殖牝馬 CSV

また、以下の CSV を書き出せます。

* 生産馬 CSV
* 種牡馬 CSV
* 繁殖牝馬 CSV

種牡馬 CSV / 繁殖牝馬 CSV のエクスポートでは、次の 2 方式を選べます。

* 史実馬のみ
* 架空馬を含む

出力ファイルは UTF-8 BOM 付きで保存されるため、Excel で開く用途も想定しています。

### 2. 産駒リスト表示

以下の単位で対象を切り替え、産駒や関連データを一覧表示できます。

* 種牡馬
* 繁殖牝馬
* 系統

確認できる内容の例:

* 所有馬
* 年度別幼駒
* 関連する繁殖牝馬
* 所属系統

### 3. グラフ表示

系統を選択し、対象期間を指定して各種グラフを確認できます。

現在対応している主な表示:

* 種牡馬一覧
* 年度別生産数
* 性別分布
* 成長型分布
* 馬場適性分布
* 距離適性分布
* 評価分布
* 河童木評価分布
* 美香評価分布
* 年度別性別比

### 4. 集計表示

種牡馬 / 繁殖牝馬 / 系統ごとの集計値を表形式で確認できます。

主な集計項目:

* 産駒数
* 所有数
* 幼駒数
* 所有率
* 性別比
* 成長型
* 馬場適性
* 距離適性
* 各種評価
* 総合評価

期間指定やソートに対応しています。

### 5. データ編集

「データ編集」画面では、年次ごとの入力・更新をしやすくするための専用画面を用意しています。

#### 新馬入厩

デビュー世代の馬について、以下を編集できます。

* 馬名
* 成長型
* 馬場適性
* 距離適性
* 評価

#### 誕生

誕生年ごとの馬について、以下を編集できます。

* 性別
* 秘書評価
* 牧場長評価
* 河童木評価
* 長峰評価
* 美香評価

#### 配合

対象年の配合データとして、以下を編集できます。

* 父
* 配合評価
* 爆発力

#### 所有繁殖牝馬

所有繁殖牝馬を牧場ごとの列に分けて表示し、ドラッグ & ドロップで所属牧場を切り替えられます。

### 6. 種牡馬マスタ編集

種牡馬マスタ画面では、以下を編集できます。

* 父
* 史実フラグ
* 系統状態

加えて、次の操作に対応しています。

* 新規追加
* CSV エクスポート
* 架空種牡馬クリーンアップ

系統状態は現在、次の値を扱います。

* `-` (系統確立なし)
* `子`
* `親`

### 7. 繁殖牝馬マスタ編集

繁殖牝馬マスタ画面では、以下を編集できます。

* 父
* 母
* 史実フラグ
* 重賞勝ちフラグ
* 牝系フラグ
* 牧場

加えて、次の操作に対応しています。

* 新規追加
* CSV エクスポート

### 8. データベース切り替え

ホーム画面の「データベースファイルを開く」から、使用する `.db` ファイルを切り替えられます。

複数のセーブデータや検証用データを分けて管理したい場合に使えます。

## CSV 仕様

### 生産馬 CSV

読み込み時に使用する主な列:

* 生年
* 名前
* 性別
* 父
* 母
* 秘書
* 牧場長
* 河童木
* 長峰
* 美香
* 成長型
* 馬場
* 距離
* 評価
* 配合
* 引退年

想定値の例:

* 性別: `牡` / `牝`
* 評価: `◎` / `○` / `▲` / `△` / `-`
* 成長型: `早熟` / `早め` / `遅め` / `覚醒` / `晩成`
* 馬場: `芝` / `ダート` / `万能`
* 距離: `短距離` / `マイル` / `中距離` / `クラシック` / `長距離`
* 配合: `S30` / `D10` / `☆` (史実馬) など

補足:

* `生年` `父` `母` `性別` が空の行は取り込まれません。
* `名前` が `☆` で始まる行は取り込まれません。
* 生産馬 CSV の取り込み後、種牡馬・繁殖牝馬マスタは生産馬データを元に補完されます。

### 種牡馬 CSV

列:

* 名前
* 父
* 史実
* 系統

`史実` は、値が入っていれば有効として扱います。

### 繁殖牝馬 CSV

列:

* 名前
* 父
* 母
* 史実
* 重賞
* 牧場

`史実` は、値が入っていれば有効として扱います。

## バックアップ

各 CSV インポートの実行前に、現在のデータは自動でバックアップ CSV として保存されます。

保存先:

```text
<Documents>\HorseDataVisualizer\backup
```

ファイル名にはタイムスタンプが付きます。

例:

```text
horses_20260410153045.csv
sires_20260410153045.csv
mares_20260410153045.csv
```

## データ保存について

* データベースには SQLite を使用しています。
* アプリ側では `.db` ファイルを開いて利用します。
* エクスポート用の内部保存先も別途作成されます。

### 起動方法

依存関係を取得:

```bash
flutter pub get
```

Drift のコード生成:

```bash
dart run build_runner build
```

監視モードを使う場合:

```bash
dart run build_runner watch
```

Windows で起動:

```bash
flutter run -d windows
```

### プロジェクト構成

```text
lib/
├─ data/
│  ├─ db/
│  │  ├─ app_database.dart
│  │  ├─ tables.dart
│  │  └─ dao/
│  ├─ entity/
│  ├─ repository/
│  └─ service/csv/
├─ presentation/
│  ├─ action/
│  ├─ misc/
│  ├─ page/
│  ├─ theme/
│  └─ widget/
└─ main.dart
```

役割の概要:

* `data/db`: Drift / SQLite の定義、DAO
* `data/entity`: 生データや集計結果のモデル
* `data/repository`: 画面から利用するデータ操作の窓口
* `data/service/csv`: CSV 読み込み処理
* `presentation/action`: ファイル操作、インポート / エクスポート
* `presentation/page`: 各画面
* `presentation/widget`: 共通 UI 部品
